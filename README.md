生态模拟游戏 (Eco-Simulation Game)
这是一个基于 C++ 和 ncurses 库开发的命令行生态模拟游戏。游戏模拟了一个简单的食物链生态系统，玩家可以选择扮演捕食者或猎物，在随机生成的草原环境中生存、进食并躲避天敌。
🎮 游戏特色
角色扮演:
老虎 (Tiger): 顶级捕食者，可以捕食狼和兔子。
狼 (Wolf): 中级捕食者，可以捕食兔子，但会被老虎捕食。
兔子 (Rabbit): 猎物，只能吃草，需要躲避狼和老虎。
动态生态系统:
草地 (Grass): 随机生成，是兔子的食物来源。
自动行为: 非玩家控制的动物会根据其本能进行移动、觅食和逃避。
生存机制:
饥饿度: 所有动物都有饥饿值，随时间减少，归零则死亡。
生命值: 受到攻击会扣除生命值。
关卡/模式: 支持不同的初始配置和游戏难度（具体由 Game 类逻辑控制）。
🛠️ 编译与运行
依赖要求
C++ 编译器 (如 g++)
ncurses 开发库 (Linux/macOS 通常预装或可通过包管理器安装，如 libncurses5-dev)
编译命令
将所有源文件编译并链接：
g++ -o eco_sim *.C -lncurses
运行游戏
./eco_sim

提示:
如果你扮演兔子，请寻找绿色的 Grass 并避开其他动物。

如果你扮演老虎，你是食物链顶端，但需注意不要让自己饿死。
🏗️ 项目结构
代码采用面向对象设计，主要类结构如下：
核心控制:
Game: 游戏主循环控制器，管理所有实体列表、碰撞检测、状态更新和输入处理。
Gui: 封装 ncurses 接口，负责屏幕渲染、颜色管理和用户输入。
基类:
Item: 所有游戏物体的基类，包含坐标、方向等基本信息。
Animal: 动物基类，继承自 Item，增加生命值、饥饿度、移动逻辑和物种类型识别。
具体实现:
动物: Tiger (老虎), Wolf (狼), Rabbit (兔子)。
环境: Grass (草地)。
通用逻辑: Predator (可能作为捕食者行为的辅助类或接口，具体视代码实现而定)。
📝 游戏机制详解
食物链逻辑:
老虎 吃 狼 和 兔子。
狼 吃 兔子。
兔子 吃 草。
当捕食者与猎物坐标重合时，触发捕食逻辑：猎物消失，捕食者饥饿度降低（或生命恢复）。
AI 行为:
捕食者: 会主动向视野内的猎物移动。
猎物: 会主动远离附近的捕食者，并向草地移动。
随机性: 在没有明确目标时，动物会随机游荡。
生存指标:
每帧游戏都会消耗动物的饥饿值。
饥饿值过低可能导致生命值下降或直接死亡。
进食是维持生存的唯一途径。
地图生成:
草地 (Grass) 会在地图上随机生成和再生，确保持续的食物供应。
📄 注意事项
游戏依赖于终端的大小，建议在较大的终端窗口中运行以获得最佳体验。
如果编译报错，请确保所有 .h 和 .C 文件都在同一目录下，并且已正确安装 ncurses 库。
